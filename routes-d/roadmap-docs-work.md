# Documentation Roadmap

This roadmap outlines planned documentation work for the Nextellar docs site. It's kept short and high-level so contributors and readers can quickly see what's coming next.

---

## Q2 2026 — Foundations

| Milestone                                                                | Status      |
| ------------------------------------------------------------------------ | ----------- |
| SDK Reference — full type and parameter docs for every exported function | In progress |
| Hooks deep-dive — expanded usage guides for all 8 hooks                  | Planned     |
| CLI command reference — flags, options, and examples for every command   | In progress |
| Error handling guide — shared recovery patterns across all hooks         | Planned     |

## Q3 2026 — Integrations & Examples

| Milestone                                                               | Status  |
| ----------------------------------------------------------------------- | ------- |
| Wallet integration walkthrough — Freighter, Albedo, Lobstr, xBull, Hana | Planned |
| Soroban smart contract tutorial — deploy, invoke, and listen to events  | Planned |
| Payment flow example — end-to-end tutorial with code                    | Planned |
| SEP compliance guide — SEP-10, SEP-24, SEP-31 walkthroughs              | Planned |

## Q4 2026 — Advanced Topics

| Milestone                                 | Status  |
| ----------------------------------------- | ------- |
| Transaction batching and fee optimization | Planned |
| Disaster recovery and key management      | Planned |
| Testing strategies for Stellar dApps      | Planned |
| Deployment and CI/CD templates            | Planned |

## Suggesting Changes

This roadmap is a living document. To suggest a change:

- Open a [GitHub issue](https://github.com/nextellarlabs/nextellar/issues) with the label `documentation`.
- Submit a pull request that updates this page directly.
- Start a discussion in [GitHub Discussions](https://github.com/nextellarlabs/nextellar/discussions) for larger proposals.

All suggestions are reviewed by the maintainers before being added.
