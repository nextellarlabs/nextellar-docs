# Running the Docs Locally

Thank you for contributing to the Nextellar documentation! This guide will help you set up and run the documentation site on your local machine so you can preview your changes before opening a pull request.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** (v18 or later)
- **pnpm** (the package manager used by this project)
- **Git**

## Setup and Run Steps

Follow these steps to get the site running locally:

### 1. Clone the repository

First, clone the `nextellar-docs` repository and navigate into the project directory:

```bash
git clone https://github.com/nextellarlabs/nextellar-docs.git
cd nextellar-docs
```

### 2. Install dependencies

Install the required dependencies using pnpm:

```bash
pnpm install
```

### 3. Build the content

We use Contentlayer to process MDX files. Before starting the development server, you must build the content:

```bash
pnpm build:content
```

### 4. Start the development server

Now you can start the Next.js development server:

```bash
pnpm dev
```

The documentation site should now be running at [http://localhost:3000](http://localhost:3000). Any changes you make to the `.mdx` files in the `docs` or `routes-d` directory will automatically hot-reload in the browser.

## Common Setup Issues

### Contentlayer errors

If you see an error related to `contentlayer2` or missing generated files, it usually means the content hasn't been built. Run `pnpm build:content` to resolve this.

### Port already in use

If port 3000 is already in use by another application, Next.js will automatically try port 3001. If it fails, you can specify a port manually:

```bash
pnpm dev --port 3001
```

### Missing dependency errors

If you encounter errors about missing modules after pulling the latest changes from the `main` branch, try clearing your `node_modules` and reinstalling:

```bash
rm -rf node_modules
pnpm install
```

## Verifying Your Changes

Before submitting your pull request, please ensure your changes meet the following criteria:

1. **Content builds without errors:** Run `pnpm build:content`
2. **Internal links are valid:** Run `pnpm check:links`
3. **Sidebar validation passes:** Run `pnpm validate:sidebar`

Once everything passes, you're ready to open a pull request!
