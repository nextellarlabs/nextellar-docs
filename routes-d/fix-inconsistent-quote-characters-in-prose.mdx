# Quote Characters in Prose

To maintain consistency across the Nextellar documentation, we have standardized the use of quote characters in all prose. 

## The Standard

**Always use straight quotes (`"` and `'`) in prose.** 

Do not use curly (smart) quotes (`“`, `”`, `‘`, `’`). 

### Examples

**Correct:**
> Install the wallet extension, refresh the page, and retry. Link users to the wallet's official install page.
> Prompt the user to try again. Do not treat this as a server failure—show a clear "Connection cancelled" message instead of a generic error.

**Incorrect:**
> Install the wallet extension, refresh the page, and retry. Link users to the wallet’s official install page.
> Prompt the user to try again. Do not treat this as a server failure—show a clear “Connection cancelled” message instead of a generic error.

## Applying the Standard

All existing occurrences of curly quotes in the `docs/` folder have been replaced with their straight quote equivalents. When writing new documentation or modifying existing pages, ensure your editor does not automatically convert straight quotes to curly quotes (a common feature in some word processors or OS-level text replacement settings).

### Code Blocks

This rule applies universally. Code blocks already use straight quotes as required by programming languages, so no changes are needed there.
