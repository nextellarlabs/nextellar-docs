# Security Policy and Vulnerability Reporting

Working draft for issue #292. Final file: `docs/guides/security-policy.mdx`.

This page describes the security policy for the Nextellar CLI, SDK, and documentation site. If you believe you have found a security vulnerability, please follow the process below.

---

## Scope

The following are in scope for security reports:

| Target | Description |
|---|---|
| `nextellar` CLI | The npm package and its install/scaffold pipeline |
| React hooks | `useStellarWallet`, `useStellarPayment`, and all other exported hooks |
| Documentation site | nextellar.dev — XSS, credential exposure, and similar web vulnerabilities |
| Scaffold templates | Code generated by `nextellar scaffold` that ships with insecure defaults |

The following are **out of scope**:

- Vulnerabilities in third-party dependencies already publicly disclosed upstream
- Issues in user-generated code or forks of Nextellar
- Social engineering attacks against maintainers
- Stellar network-level issues — report those to the [Stellar Development Foundation](https://www.stellar.org/bug-bounty)

---

## Reporting Channels

**Do not open a public GitHub issue for security vulnerabilities.**

| Channel | Address | Use for |
|---|---|---|
| GitHub Security Advisories | [Report a vulnerability](https://github.com/nextellarlabs/nextellar/security/advisories/new) | Preferred |
| Email | security@nextellar.dev | If GitHub isn't suitable |

Include: description, affected component/version, reproduction steps, and impact assessment.

---

## Response Expectations

| Stage | Target timeframe |
|---|---|
| Acknowledgement | Within 2 business days |
| Triage | Within 5 business days |
| Fix plan shared | 14 days (critical/high) / 30 days (medium/low) |
| Public disclosure | After fix, coordinated with reporter |

---

## Coordinated Disclosure

Nextellar follows coordinated disclosure with a **90-day embargo** from initial report or until a fix is released, whichever comes first. Full details are published as a [GitHub Security Advisory](https://github.com/nextellarlabs/nextellar/security/advisories) after the fix ships.
