# Fix Mismatched Code Block Fences

## Issue Summary

Several code blocks across the documentation had mismatched opening and closing fences. Specifically, blocks opened with ``bash` or ``tsx` were incorrectly closed with ````text`.

## Resolution

- Identified all `.md` and `.mdx` files containing the mismatched closing fence (````text`).
- Replaced the mismatched ``text` with the standard closing fence ``.
- Verified the integrity of the files by running the contentlayer build script (`pnpm build:content`).

## Verification

All code blocks now render properly, and the MDX parser successfully processes the files without throwing structural mismatch errors.
