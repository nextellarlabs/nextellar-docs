# Common Error Messages

This reference lists error messages you are likely to see when scaffolding a Nextellar app, connecting a wallet, sending payments, or contributing to the documentation. Each entry includes the message (or pattern), a likely cause, and a recommended fix.

> **Tip:** Copy the exact error text from your terminal or browser console when searching this page or opening a GitHub issue.

---

## CLI and installation

### `nextellar: command not found`

|                  |                                                                                                                                                 |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| **Likely cause** | The CLI is not installed globally, or your shell `PATH` does not include the global npm bin directory.                                          |
| **Fix**          | Run without a global install: `npx nextellar my-app`. To install globally: `npm install -g nextellar`, then confirm `which nextellar` resolves. |

### `EACCES: permission denied`

|                  |                                                                                                                                                                        |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Likely cause** | npm tried to write to a system-owned directory during global install or cache operations.                                                                              |
| **Fix**          | Prefer `npx nextellar my-app` (no global install). Or fix ownership: `sudo chown -R $(whoami) ~/.npm`. Avoid using `sudo` with `npm install` for project dependencies. |

### Installation stops or times out during scaffold

|                  |                                                                                                                                                                                                                                  |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Likely cause** | Slow network, registry timeouts, or low disk space while dependencies install.                                                                                                                                                   |
| **Fix**          | Increase the timeout: `npx nextellar my-app --install-timeout 3600000`. Or skip auto-install and install manually: `npx nextellar my-app --skip-install` then `cd my-app && npm install`. Ensure at least 500MB free disk space. |

### Unsupported Node version / scaffold fails before install

|                  |                                                                                                              |
| ---------------- | ------------------------------------------------------------------------------------------------------------ |
| **Likely cause** | Node.js is older than the version Nextellar requires (20.18.0+).                                             |
| **Fix**          | Run `node --version` and upgrade with [nodejs.org](https://nodejs.org/) or a version manager (`nvm`, `fnm`). |

### `Port 3000 is already in use`

|                  |                                                                                                           |
| ---------------- | --------------------------------------------------------------------------------------------------------- |
| **Likely cause** | Another dev server or process is bound to port 3000.                                                      |
| **Fix**          | Start on another port: `npm run dev -- -p 3001`. Stop the other process or close the app using port 3000. |

### `Module not found` / dependency resolution errors

|                  |                                                                                                                                                                                                  |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Likely cause** | Incomplete install, corrupted `node_modules`, or mixed package managers.                                                                                                                         |
| **Fix**          | From the project root: `rm -rf node_modules package-lock.json && npm install` (or use `pnpm`/`yarn` consistently with the lockfile in the repo). Re-scaffold only if the project tree is broken. |

### TypeScript or `npm run build` errors after install

|                  |                                                                                                                                             |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| **Likely cause** | Dependencies not fully installed, or version mismatch after a partial install.                                                              |
| **Fix**          | Reinstall dependencies (see above), then run `npm run build` again. Confirm you used the same package manager the project was created with. |

## Wallet connection

### `User rejected` / `User rejected the request`

|                  |                                                                                                                                             |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| **Likely cause** | The user closed the wallet popup or declined the connection or signature request.                                                           |
| **Fix**          | Prompt the user to try again. Do not treat this as a server failureâ€”show a clear "Connection cancelled" message instead of a generic error. |

### `not installed` / wallet extension not found

|                  |                                                                                                                                                                      |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Likely cause** | The selected wallet (for example Freighter) is not installed in the browser or is disabled.                                                                          |
| **Fix**          | Install the wallet extension, refresh the page, and retry. Link users to the wallet's official install page (see [Wallet Integrations](/docs/integrations/wallets)). |

### Popup blocked / connection timeout

|                  |                                                                                                            |
| ---------------- | ---------------------------------------------------------------------------------------------------------- |
| **Likely cause** | Browser blocked the wallet popup, or the extension is locked.                                              |
| **Fix**          | Allow popups for `localhost` (or your app origin). Unlock the wallet extension and retry `Connect Wallet`. |

### Wallet connection fails (no specific message)

|                  |                                                                                                                                                                                                                          |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Likely cause** | Wrong network in the wallet (mainnet vs testnet), locked wallet, or stale page state.                                                                                                                                    |
| **Fix**          | Match the wallet network to your app (`TESTNET` for development). Unlock the wallet, hard-refresh the page, and try again. See [Quick Start troubleshooting](/docs/getting-started/quick-start#wallet-connection-fails). |

## Stellar network and transactions

### `op_underfunded` / insufficient balance

|                  |                                                                                                                                                                                                       |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Likely cause** | The source account does not have enough XLM (or asset balance) for the payment plus reserves and fees.                                                                                                |
| **Fix**          | Check balances with [useStellarBalances](/docs/hooks/use-stellar-balances). On testnet, fund the account via Friendbot. Ensure amount is greater than 0 and leave enough XLM for the minimum reserve. |

### `tx_bad_seq`

|                  |                                                                                                                                             |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| **Likely cause** | Transaction sequence number is out of sync with the account on the network (often after a failed or concurrent submit).                     |
| **Fix**          | Rebuild and resubmit the transaction. Avoid submitting the same signed XDR twice; fetch a fresh sequence from Horizon before signing again. |

### `tx_too_late`

|                  |                                                                                                    |
| ---------------- | -------------------------------------------------------------------------------------------------- |
| **Likely cause** | The transaction expired before it was included in a ledger (timebound too tight or network delay). |
| **Fix**          | Rebuild the transaction with a new timebound and submit again promptly.                            |

### Horizon `404` / account not found

|                  |                                                                                                                                                                                    |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Likely cause** | The public key has never been funded on the selected network, or the wrong Horizon URL/network is configured.                                                                      |
| **Fix**          | Fund the account on testnet (Friendbot) or verify `NEXT_PUBLIC_HORIZON_URL` and `NEXT_PUBLIC_NETWORK` match. See [Horizon integration](/docs/integrations/horizon#error-handling). |

### Horizon `400` / bad request

|                  |                                                                                                                                                                                      |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Likely cause** | Invalid parameters in the Horizon request (malformed address, asset, or query).                                                                                                      |
| **Fix**          | Validate addresses (56-character keys starting with `G`) and amounts before calling hooks. Check hook inputs in [useStellarPayment](/docs/hooks/use-stellar-payment#error-handling). |

### Horizon `504` / timeout

|                  |                                                                                                                                                |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| **Likely cause** | Horizon or the network is slow or unreachable.                                                                                                 |
| **Fix**          | Retry with `refresh()` on the hook. For production, use a reliable Horizon endpoint or self-host Horizon. Reduce aggressive polling intervals. |

### Payment fails / invalid recipient

|                  |                                                                                                                                                    |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Likely cause** | Recipient address format is wrong, amount is invalid, or network mismatch.                                                                         |
| **Fix**          | Use a valid Stellar address (`Gâ€¦`, 56 characters). Confirm testnet vs mainnet. See [Quick Start](/docs/getting-started/quick-start#payment-fails). |

### Transaction not appearing in history

|                  |                                                                                                                                                                                                   |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Likely cause** | Ledger not yet closed, wrong network explorer, or polling delay.                                                                                                                                  |
| **Fix**          | Wait 5â€“10 seconds and refresh. Confirm testnet vs mainnet on [Stellar Expert](https://stellar.expert). Use [useTransactionHistory](/docs/hooks/use-transaction-history) with appropriate polling. |

## Configuration and environment

### `NEXT_PUBLIC_HORIZON_URL is required`

|                  |                                                                                                                                                                          |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Likely cause** | Required environment variable is missing at build or runtime.                                                                                                            |
| **Fix**          | Add `NEXT_PUBLIC_HORIZON_URL` to `.env.local` (see [CLI Configuration](/docs/cli/configuration#environment-variables)). Restart the dev server after changing env files. |

### `NEXT_PUBLIC_NETWORK is required`

|                  |                                                                                               |
| ---------------- | --------------------------------------------------------------------------------------------- |
| **Likely cause** | Network passphrase env var is not set.                                                        |
| **Fix**          | Set `NEXT_PUBLIC_NETWORK` to `TESTNET` or `PUBLIC` in `.env.local` and restart `npm run dev`. |

### Wrong network / mainnet vs testnet mismatch

|                  |                                                                                                                    |
| ---------------- | ------------------------------------------------------------------------------------------------------------------ |
| **Likely cause** | Wallet, Horizon URL, and `NEXT_PUBLIC_NETWORK` point at different networks.                                        |
| **Fix**          | Update all three together. See [CLI Configuration â€” network switching](/docs/cli/configuration#network-switching). |

## Documentation contributors (this repo)

### `Missing required field "description"`

|                  |                                                                                                                                                             |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Likely cause** | MDX file is missing required Contentlayer frontmatter.                                                                                                      |
| **Fix**          | Add `title` and `description` to the YAML block at the top of the file. See [Contributing](/docs/guides/contributing#troubleshooting-content-build-issues). |

### `Link target not found` / broken internal link

|                  |                                                                                                                         |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------- |
| **Likely cause** | Markdown link points to a path that does not exist under `docs/`.                                                       |
| **Fix**          | Use paths like `/docs/cli/commands` and confirm the target file exists. Run `pnpm check:links` with `pnpm dev` running. |

### `Unexpected token` / MDX syntax error

|                  |                                                                                                                                    |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| **Likely cause** | Unclosed JSX tag, invalid MDX, or unescaped angle brackets in prose.                                                               |
| **Fix**          | Close all JSX components. Wrap literal angle-bracket patterns in backticks. Run `pnpm build:content` for the file and line number. |

### `Can't resolve 'contentlayer/generated'`

|                  |                                                                                                                    |
| ---------------- | ------------------------------------------------------------------------------------------------------------------ |
| **Likely cause** | Content build has not run or failed, so `.contentlayer` was not generated.                                         |
| **Fix**          | Run `pnpm build:content` successfully, then `pnpm dev`. Clear cache: `rm -rf .contentlayer && pnpm build:content`. |
