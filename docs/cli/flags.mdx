# Nextellar CLI Flags & Options

This page collects every Nextellar CLI flag into a single reference table.
Flags are grouped by category and include default values where relevant.

Use the table below to find the exact flag name, alias, default, and a short description.

| Category     | Flag                     | Alias | Default                               | Description                                                                  |
| ------------ | ------------------------ | ----- | ------------------------------------- | ---------------------------------------------------------------------------- |
| Language     | `--typescript`           | `-t`  | `true`                                | Generate a TypeScript project (default).                                     |
| Language     | `--javascript`           | `-j`  | `false`                               | Generate a JavaScript project instead of TypeScript; planned future support. |
| Network      | `--horizon-url <url>`    |       | `https://horizon-testnet.stellar.org` | Override the default Horizon API endpoint.                                   |
| Network      | `--soroban-url <url>`    |       | `https://soroban-testnet.stellar.org` | Override the default Soroban RPC endpoint.                                   |
| Wallet       | `--wallets <list>`       | `-w`  | `freighter,albedo,lobstr`             | Comma-separated list of wallet adapters to include.                          |
| Template     | `--example <name>`       | `-e`  | none                                  | Use a named example template for the generated project.                      |
| Installation | `--skip-install`         |       | `false`                               | Generate files but skip automatic dependency installation.                   |
| Installation | `--package-manager <pm>` |       | Auto-detected                         | Choose `npm`, `yarn`, or `pnpm` for install.                                 |
| Installation | `--install-timeout <ms>` |       | `1200000`                             | Set dependency install timeout in milliseconds.                              |
| Automation   | `--defaults`             | `-d`  | `false`                               | Skip prompts and use default settings.                                       |
| Feature Add  | `--list`                 | `-l`  | `false`                               | List available features for `nextellar add`.                                 |
| Feature Add  | `--force`                | `-f`  | `false`                               | Force feature addition, overwriting existing files.                          |
| Deployment   | `--dry-run`              |       | `false`                               | Validate deployment without creating a final bundle.                         |
| Information  | `--help`                 | `-h`  | `false`                               | Show CLI help text.                                                          |
| Information  | `--version`              | `-v`  | `false`                               | Print the installed Nextellar CLI version.                                   |

## How to use this table

Most options are used with the primary scaffold command:

```bash
npx nextellar my-app --wallets freighter,xbull --package-manager pnpm --skip-install
```

Feature add options are used with `nextellar add`:

```bash
nextellar add freighter --force --package-manager pnpm
```

Deployment options are used with `nextellar deploy`:

```bash
nextellar deploy --dry-run
```

## Notes

- The `Alias` column shows shorthand forms where supported.
- Default values are provided when the CLI exposes a standard fallback.
- Use `--help` or `-h` to confirm available flags in your installed version.
