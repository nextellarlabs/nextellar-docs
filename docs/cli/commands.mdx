# CLI Commands

Complete reference for all Nextellar CLI commands and their usage.

## Main Command

### nextellar

Create a new Stellar dApp project.

````bash
nextellar <project-name> [options]
```css

**Arguments:**

| Argument       | Type   | Required | Description                    |
| -------------- | ------ | -------- | ------------------------------ |
| `project-name` | string | Yes      | Name of your project directory |

**Examples:**

```bash
# Basic usage
nextellar my-stellar-app

# With custom package manager
nextellar my-app --package-manager pnpm

# Skip installation
nextellar my-app --skip-install

# Full customization
nextellar my-app \
  --package-manager yarn \
  --horizon-url https://horizon.stellar.org \
  --wallets freighter,xbull \
  --skip-install
```css

## Help Commands

### --help, -h

Display help information.

```bash
nextellar --help
# or
nextellar -h
```

**Output:**

```bash
Usage: nextellar <project-name> [options]

Options:
  -t, --typescript         Generate a TypeScript project (default)
  -j, --javascript         Generate a JavaScript project
  --horizon-url <url>      Override default Horizon endpoint
  --soroban-url <url>      Override default Soroban RPC endpoint
  -w, --wallets <list>     Comma-separated list of wallet adapters
  -d, --defaults           Skip prompts and use defaults
  --skip-install           Skip dependency installation
  --package-manager <pm>   Choose package manager (npm, yarn, pnpm)
  --install-timeout <ms>   Installation timeout in milliseconds
  -v, --version            Show CLI version
  -h, --help               Show help text
```css

### --version, -v

Display CLI version.

```bash
nextellar --version
# or
nextellar -v
```

**Output:**

```bash
1.0.4
```

## Feature Commands

### nextellar add

Add Stellar features to an existing Nextellar project. This command allows you to incrementally add wallet adapters, smart contract support, or pre-built UI components as your application grows.

```bash
nextellar add <feature-name> [options]
```

**Arguments:**

| Argument       | Type   | Required | Description                     |
| -------------- | ------ | -------- | ------------------------------- |
| `feature-name` | string | No       | Name of the feature to add      |

**Options:**

| Option                  | Alias  | Description                                                     |
| ----------------------- | ------ | --------------------------------------------------------------- |
| `--list`                | `-l`   | List all available features that can be added                   |
| `--force`               | `-f`   | Force add feature, overwriting existing files without prompting |
| `--skip-install`        |        | Skip automatic dependency installation after adding the feature |
| `--package-manager <pm>`|        | Choose package manager (npm, yarn, pnpm) for installation      |

**Examples:**

```bash
# List all available features
nextellar add --list

# Add a Soroban smart contract feature to the project
nextellar add soroban

# Force add Freighter wallet support, overwriting existing configuration
nextellar add freighter --force --package-manager pnpm
```

### nextellar doctor

Run environment diagnostics to verify your machine is ready for Nextellar development. The doctor command checks local tooling, network connectivity to Stellar endpoints, and available system resources.

```bash
nextellar doctor [options]
```

**Options:**

| Option   | Description                                      |
| -------- | ------------------------------------------------ |
| `--json` | Output results as JSON for CI integration        |

**What it checks:**

| Check          | Required | Description                                              |
| -------------- | -------- | -------------------------------------------------------- |
| Node.js        | Yes      | Version 20 or later                                      |
| npm            | Yes      | Package manager bundled with Node.js                     |
| yarn           | No       | Optional package manager                                 |
| pnpm           | No       | Optional package manager                                 |
| Git            | Yes      | Version control for project scaffolding                  |
| Rust           | No       | Needed for Soroban contract development                  |
| Stellar CLI    | No       | Needed for contract deployment and testing               |
| wasm32 target  | No       | Rust target for compiling Soroban contracts              |
| Horizon API    | Yes      | Reachability of the default testnet Horizon endpoint     |
| Soroban RPC    | No       | Reachability of the default testnet Soroban RPC endpoint |
| Free memory    | Yes      | At least 1 GB of available RAM                           |

**Examples:**

```bash
# Run environment diagnostics
nextellar doctor

# Output results as JSON (useful in CI pipelines)
nextellar doctor --json
```

**Standard output:**

```bash

### nextellar upgrade

Upgrade an existing Nextellar project to the latest template files.

```bash
nextellar upgrade [options]
```

Run this command from the root of an existing Nextellar project. Use a dry run first to review the files that would change, then run the upgrade when you are ready to apply updates.

**Options:**

| Option      | Description                                           |
| ----------- | ----------------------------------------------------- |
| `--dry-run` | Show what would change without applying the upgrade   |
| `--yes`     | Apply changes without prompting for confirmation      |

**Examples:**

```bash
# Preview upgrade changes
nextellar upgrade --dry-run

# Apply the upgrade interactively
nextellar upgrade

# Apply the upgrade without prompting
nextellar upgrade --yes
```

### nextellar deploy

Validate and prepare your project for production deployment. This command checks that your Horizon and Soroban RPC endpoints are properly configured, and builds a optimized production bundle of your Stellar dApp.

```bash
nextellar deploy [options]
```

**Options:**

| Option       | Description                                                                 |
| ------------ | --------------------------------------------------------------------------- |
| `--dry-run`  | Validate configuration and build process without producing a deploy bundle |

**Examples:**

```bash
# Prepare project for deployment
nextellar deploy

# Run a dry-run validation without actual deployment
nextellar deploy --dry-run
```

## Project Creation Workflow

When you run `nextellar my-app`, here's what happens:

### Step 1: Validation

- Checks if project name is valid.
- Verifies directory doesn't already exist.
- Validates CLI options.

### Step 2: Scaffolding

- Creates project directory.
- Copies template files.
- Substitutes configuration variables.
- Generates package.json.

### Step 3: Installation

- Detects package manager (npm/yarn/pnpm).
- Installs dependencies.
- Verifies installation success.

### Step 4: Success

- Displays success message.
- Shows next steps.
- Provides helpful links.

## Exit Codes

| Code | Meaning             | Description                       |
| ---- | ------------------- | --------------------------------- |
| `0`  | Success             | Project created successfully      |
| `1`  | Error               | General error (see error message) |
| `2`  | Invalid Arguments   | Invalid command-line arguments    |
| `3`  | Directory Exists    | Project directory already exists  |
| `4`  | Installation Failed | Dependency installation failed    |

## Command Examples

### Basic Project

```bash
npx nextellar stellar-payment-app
```

Creates a project with default settings:

- TypeScript enabled
- testnet configuration
- Default wallets (all supported)
- Automatic dependency installation

### Production Configuration

```bash
npx nextellar stellar-mainnet-app \
  --horizon-url https://horizon.stellar.org \
  --soroban-url https://soroban-rpc.stellar.org \
  --package-manager pnpm
```

Creates a mainnet-ready project.

### Development Setup

```bash
npx nextellar dev-app \
  --skip-install \
  --wallets freighter
```

Creates a project for development:

- Skips installation (install manually later)
- Only includes Freighter wallet

### Custom Timeout

```bash
npx nextellar my-app --install-timeout 3600000
```

Increases installation timeout to 1 hour (useful for slow networks).

## Interactive Mode

Nextellar can prompt for scaffold configuration when it runs in a suitable interactive terminal.

Prompts are shown when standard input and output are both TTYs, the `CI` environment variable is not set, and `--defaults` is not provided. If any of those conditions are not met, the CLI uses provided flags and default values instead.

Flags still take precedence over prompts. For example, passing `--wallets`, `--horizon-url`, `--soroban-url`, `--package-manager`, or `--skip-install` keeps those values fixed and skips prompting for that setting.

## Command Aliases

Nextellar doesn't currently support command aliases, but you can create your own:

**Bash/Zsh:**

```bash
# Add to ~/.bashrc or ~/.zshrc
alias nxt="npx nextellar"

# Usage
nxt my-app
```

**PowerShell:**

```powershell
# Add to $PROFILE
function nxt { npx nextellar $args }

# Usage
nxt my-app
```bash
````
