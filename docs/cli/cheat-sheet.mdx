# CLI Cheat Sheet

Quick reference for common Nextellar CLI commands and development workflow. Bookmark this page for fast access during development.

## Project Creation

### Basic Project Setup

```bash
# Create new Stellar dApp
nextellar my-stellar-app

# Create with custom package manager
nextellar my-app --package-manager pnpm

# Create without installing dependencies
nextellar my-app --skip-install

# Create with specific wallets only
nextellar my-app --wallets freighter,xbull
```

### Advanced Project Creation

```bash
# Full customization
nextellar my-app \
  --package-manager yarn \
  --horizon-url https://horizon.stellar.org \
  --wallets freighter,albedo,lobstr \
  --skip-install

# Use defaults (skip all prompts)
nextellar my-app --defaults
```

## Development Commands

### Content & Build

```bash
# Build content layer (required after adding/editing MDX)
pnpm build:content

# Start development server
pnpm dev

# Full production build
pnpm build

# Start production server
pnpm start
```

### Code Quality

```bash
# Format code with Prettier
pnpm format

# Check if code is formatted
pnpm format:check

# Run ESLint
pnpm lint

# Run tests
pnpm test
```

### Documentation Validation

```bash
# Validate internal links
pnpm check:links

# Validate sidebar configuration
pnpm validate:sidebar

# Run all validation checks
pnpm build:content && pnpm check:links
```

## Feature Management

### Adding Features

```bash
# List all available features
nextellar add --list

# Add Soroban smart contract support
nextellar add soroban

# Add wallet adapter
nextellar add freighter

# Force add (overwrite existing files)
nextellar add soroban --force
```

### Feature Examples

```bash
# Add multiple wallet adapters
nextellar add wallets --wallets freighter,albedo,xbull

# Add UI components
nextellar add components

# Add testing setup
nextellar add testing
```

## Deployment

### Production Preparation

```bash
# Validate and prepare for deployment
nextellar deploy

# Dry run (validate without deploying)
nextellar deploy --dry-run

# Deploy with specific configuration
nextellar deploy --env production
```

### Build Verification

```bash
# Verify production build works
pnpm build && pnpm start

# Check bundle size
pnpm build && npx @next/bundle-analyzer
```

## Troubleshooting

### Common Issues

```bash
# Clear content cache
rm -rf .contentlayer && pnpm build:content

# Clear Next.js cache
rm -rf .next && pnpm build

# Reinstall dependencies
rm -rf node_modules pnpm-lock.yaml && pnpm install

# Reset git state
git reset --hard HEAD
git clean -fd
```

### Debugging

```bash
# Run with verbose output
DEBUG=* pnpm dev

# Check for TypeScript errors
pnpm tsc --noEmit

# Validate package.json
pnpm audit

# Check for outdated packages
pnpm outdated
```

## Quick Workflows

### Adding New Documentation

```bash
# 1. Create new MDX file
touch docs/guides/my-new-guide.mdx

# 2. Add frontmatter and content
# (edit file with your preferred editor)

# 3. Build content
pnpm build:content

# 4. Validate
pnpm check:links

# 5. Test locally
pnpm dev
```

### Fixing Broken Links

```bash
# 1. Find broken links
pnpm check:links

# 2. Fix links in files
# (edit files to correct link targets)

# 3. Validate fix
pnpm check:links

# 4. Rebuild content
pnpm build:content
```

### Preparing for PR

```bash
# Complete validation workflow
pnpm format:check && \
pnpm lint && \
pnpm build:content && \
pnpm check:links && \
pnpm build
```

## Package Manager Commands

### pnpm (Recommended)

```bash
pnpm install          # Install dependencies
pnpm add <package>    # Add dependency
pnpm remove <package> # Remove dependency
pnpm update           # Update dependencies
pnpm run <script>     # Run package.json script
```

### npm Alternative

```bash
npm install           # Install dependencies
npm install <package> # Add dependency
npm uninstall <package> # Remove dependency
npm update            # Update dependencies
npm run <script>      # Run package.json script
```

### yarn Alternative

```bash
yarn install          # Install dependencies
yarn add <package>    # Add dependency
yarn remove <package> # Remove dependency
yarn upgrade          # Update dependencies
yarn run <script>     # Run package.json script
```
