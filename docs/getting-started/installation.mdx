# Installation Guide

<Steps>
  <Step>
    <StepTitle>Install globally (recommended for frequent use)</StepTitle>
    <StepContent>
      <div>
        <p>Install the Nextellar CLI globally so it is available from any terminal session.</p>
```bash
# Using npm
npm install -g nextellar

# Using pnpm
pnpm add -g nextellar

# Using yarn
yarn global add nextellar
```

        <p>After installing, scaffold a new project with:</p>
```bash
nextellar my-app
```

        <p>Note: we recommend using a Node version manager (nvm, fnm, volta) to avoid permission issues.</p>
      </div>
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Use on-demand runner (no global install)</StepTitle>
    <StepContent>
      <div>
        <p>Run the CLI without a global install using one of the following one-off commands:</p>

        <CodeTabs tabs={{
          "npx": { "syntax": "npx nextellar my-app" },
          "pnpm": { "syntax": "pnpm dlx nextellar my-app" },
          "bun": { "syntax": "bunx nextellar my-app" }
        }} />

        <p>These commands download and run the CLI for a single invocation and are ideal for quick trials.</p>
      </div>
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Scaffold a new project</StepTitle>
    <StepContent>
      <div>
        <p>Scaffold a Nextellar project (global install or on-demand runner):</p>
```bash
npx nextellar my-app

# or, if installed globally
nextellar my-app
```

        <p>By default the scaffold includes a Next.js + TypeScript app, Stellar adapters, UI components, and installs dependencies. Use <code>--skip-install</code> to skip automatic dependency installation.</p>
      </div>
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Change into the project & run</StepTitle>
    <StepContent>
      <div>
        <p>After scaffolding, change into the project directory and start the dev server:</p>
```bash
cd my-app

# npm
npm run dev

# pnpm
pnpm dev

# yarn
yarn dev
```

        <p>Open <code>http://localhost:3000</code> to view the app.</p>
      </div>
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Verify installation</StepTitle>
    <StepContent>
      <div>
        <p>Confirm the CLI is available and check the version:</p>
```bash
nextellar --version
nextellar --help
```

        <p>For on-demand runners, prefix with the runner (e.g. <code>npx nextellar --version</code>).</p>
      </div>
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Common flags</StepTitle>
    <StepContent>
      <div>
        <p>Useful flags when scaffolding:</p>
```bash
npx nextellar my-app --skip-install
npx nextellar my-app --js
npx nextellar my-app --horizon-url https://horizon.stellar.org
npx nextellar my-app --soroban-url https://rpc-futurenet.stellar.org
npx nextellar my-app --wallets freighter,xbull,ledger
```
      </div>
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Troubleshooting & tips</StepTitle>
    <StepContent>
      <div>
        <ul>
          <li><strong>Permissions</strong>: Use a Node version manager to avoid <code>sudo</code> for global installs.</li>
          <li><strong>Node version</strong>: Ensure Node 18+ is active (<code>node -v</code>).</li>
          <li><strong>Skip install</strong>: Use <code>--skip-install</code> if you want to manage deps manually.</li>
          <li><strong>Network</strong>: Clear package manager cache if dependency installs fail.</li>
        </ul>
      </div>
    </StepContent>
  </Step>
</Steps>
