# Quick Start

This quick start walks you through scaffolding a new Nextellar project, running it locally, and inspecting the generated scaffold so you can begin building Stellar-powered features immediately.

<Steps>
  <Step>
    <StepTitle>Step 1: Scaffold a new project</StepTitle>
    <StepContent>
      <div>
        <p>Run the following command to scaffold a new Nextellar application. We recommend using the on-demand runner so you don't need a global install:</p>
```bash
npx nextellar my-app
```

        <p>The generator creates a Next.js + TypeScript app with Stellar integrations, common UI components, and a set of sensible defaults. Pass <code>--skip-install</code> to generate files without installing dependencies.</p>
      </div>
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Step 2: Change into the project</StepTitle>
    <StepContent>
      <div>
        <p>After scaffolding, change into the generated directory:</p>
```bash
cd my-app
```

        <p>Inspect <code>package.json</code> to confirm which package manager was selected and which scripts are available.</p>
      </div>
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Step 3: Install dependencies (if skipped)</StepTitle>
    <StepContent>
      <div>
        <p>If you used <code>--skip-install</code>, install dependencies now with the package manager of your choice:</p>
```bash
# npm
npm install

# pnpm
pnpm install

# yarn
yarn install
```
      </div>
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Step 4: Run the development server</StepTitle>
    <StepContent>
      <div>
        <p>Start the dev server using the script in the scaffolded project:</p>

        <CodeTabs tabs={{
          "npm": { "syntax": "npm run dev" },
          "pnpm": { "syntax": "pnpm dev" },
          "yarn": { "syntax": "yarn dev" }
        }} />

        <p>By default the site is served at <code>http://localhost:3000</code>. Visit the URL and verify the sample landing page and wallet connect UI render correctly.</p>
      </div>
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Step 5: Connect a Stellar wallet</StepTitle>
    <StepContent>
      <div>
        <p>Use the built-in wallet connection UI to connect a supported Stellar wallet (Freighter, XBull, Ledger, etc.).</p>

        <p>Once connected you should see:</p>
        <ul>
          <li>Account address and network</li>
          <li>Balances and trustlines</li>
          <li>Recent transactions and statuses</li>
        </ul>

        <p>The wallet UI and hooks are located under <code>components/</code> and <code>hooks/</code> in the scaffold.</p>
      </div>
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Step 6: Explore the project structure</StepTitle>
    <StepContent>
      <div>
        <p>The generated scaffold uses a clear, scalable layout. Key folders to inspect:</p>

        <FolderTree className="overflow-hidden p-2">
          <Folder element="my-app" defaultOpen={true}>
            <Folder element="app" defaultOpen={true}>
              <File><p>page.tsx</p></File>
              <File><p>layout.tsx</p></File>
            </Folder>
            <Folder element="components">
              <File><p>WalletConnect.tsx</p></File>
              <File><p>AccountBalance.tsx</p></File>
            </Folder>
            <Folder element="hooks">
              <File><p>useStellarAccount.ts</p></File>
              <File><p>useStellarPayment.ts</p></File>
            </Folder>
            <Folder element="lib">
              <Folder element="stellar">
                <File><p>client.ts</p></File>
                <File><p>soroban.ts</p></File>
              </Folder>
            </Folder>
            <File><p>package.json</p></File>
            <File><p>tailwind.config.js</p></File>
          </Folder>
        </FolderTree>
      </div>
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Step 7: Run tests</StepTitle>
    <StepContent>
      <div>
        <p>Run unit and integration tests provided in the scaffold using the test script:</p>
```bash
npm test
# or
pnpm test
# or
yarn test
```

        <p>The scaffold includes Jest + React Testing Library configurations.</p>
      </div>
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Step 8: Build & deploy</StepTitle>
    <StepContent>
      <div>
        <p>Build the production bundle and deploy to your preferred hosting provider:</p>
```bash
# build
npm run build

# start (production)
npm start
```

        <p>The scaffold includes example GitHub Actions workflow files you can adapt for automated deployments.</p>
      </div>
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Step 9: Customize</StepTitle>
    <StepContent>
      <div>
        <p>Customize the scaffold to suit your product:</p>
        <ul>
          <li>Replace or style components in <code>components/</code>.</li>
          <li>Extend hooks in <code>hooks/</code> to add app-specific logic.</li>
          <li>Integrate additional wallets or services via new adapters.</li>
        </ul>
      </div>
    </StepContent>
  </Step>
</Steps>

---

You're ready to build on top of the generated scaffold. For in-depth guides on wallet adapters, Soroban contracts, and advanced configuration, see the corresponding reference pages in this documentation.
