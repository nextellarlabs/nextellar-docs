# Redirects Map

When a docs page moves, keep the old URL alive long enough for bookmarks, search results, and external links to keep working. This page documents the redirect strategy used for the site.

## Principles

- Prefer permanent redirects for moved pages.
- Redirect old URLs to the closest canonical replacement.
- Keep redirect targets within the docs site when possible.
- Update the redirect map whenever a page is renamed or reorganized.

| `/docs/hooks/errors` | `/docs/guides/hook-error-handling` | Legacy landing page for hook failures. |

## How to maintain the map

1. Add the new canonical page first.
2. Record the old URL and the replacement URL in this table.
3. Add the runtime redirect in the app layer when the move is user-facing.
4. Verify the old path still resolves after deployment.

## Notes

- Redirects are part of the documentation contract, not just a deployment detail.
- If two pages merge, redirect the weaker page to the stronger canonical page.
- If a section is removed entirely, redirect to the closest conceptual replacement instead of a 404.
