# Link Validation Scripts

Nextellar docs ships three validation scripts in the `scripts/` directory. Run them before opening a pull request to catch broken links and stale sidebar entries early.

## Scripts overview

| Script                         | npm command             | What it checks                                   |
| ------------------------------ | ----------------------- | ------------------------------------------------ |
| `scripts/validate-sidebar.cjs` | `pnpm validate:sidebar` | Sidebar and TOC entries against actual MDX files |
| `scripts/check-links.cjs`      | `pnpm check:links`      | Component page URLs against a running dev server |
| `scripts/final-validation.cjs` | _(run directly)_        | Acceptance criteria summary for PR review        |

## validate-sidebar.cjs

Compares every `/docs/components/*` href in `config/sidebar.tsx` and `config/toc.tsx` against the MDX files that actually exist in `docs/components/`.

### Run it

```bash
pnpm validate:sidebar
```

### What it does

1. Reads all `.mdx` files in `docs/components/` — these are the source of truth.
2. Extracts every `/docs/components/<slug>` href from `config/sidebar.tsx` and `config/toc.tsx`.
3. Reports broken claims (hrefs with no matching file) and valid claims.
4. Writes a `sidebar-validation.json` snapshot used by the other scripts.

### Reading the output

```
🔍 COMPONENTS INVENTORY (SOURCE OF TRUTH)
VALID (2): connect-wallet-button, use-window-size

📋 SIDEBAR CLAIMS
CLAIMS (2): connect-wallet-button, use-window-size

❌ BROKEN SIDEBAR LINKS (0):

✅ VALID SIDEBAR LINKS (2):
  - connect-wallet-button
  - use-window-size

✅ SUMMARY
Sidebar: 2/2 valid (100%)
TOC: 2/2 valid (100%)
Total broken: 0

🎉 ALL COMPONENT REFERENCES ARE VALID!
```

If broken links are found, the script exits with code `1` and lists the offending slugs. Remove or fix those entries in `config/sidebar.tsx` before proceeding.

## check-links.cjs

Makes live HTTP requests to a running dev server and verifies that each component page returns HTTP 200.

### Prerequisites

The dev server must be running before you execute this script:

```bash
pnpm dev
```

### Run it

```bash
pnpm check:links
```

### What it does

1. Reads `sidebar-validation.json` (generated by `validate-sidebar.cjs`).
2. Filters to only the sidebar's claimed links that are not already flagged as broken.
3. Sends an HTTP GET to `http://localhost:3000/docs/components/<slug>` for each link.
4. Reports the HTTP status code for every URL.

### Reading the output

```
🔗 TESTING COMPONENT LINKS

Testing 2 links...

[1/2] ✅ http://localhost:3000/docs/components/connect-wallet-button → 200
[2/2] ✅ http://localhost:3000/docs/components/use-window-size → 200

📊 RESULTS
✅ Passed: 2/2
❌ Failed: 0/2

🎉 ALL LINKS VALID!
```

A `❌` line means the page returned a non-200 status or the request failed entirely. Check that the MDX file exists, the frontmatter is valid, and the dev server rebuilt successfully after your last change.

If the script exits with an error about the server not being reachable, confirm `pnpm dev` is running and listening on port 3000.

## final-validation.cjs

Produces a human-readable acceptance criteria summary intended for PR descriptions and reviewer sign-off.

### Run it

```bash
node scripts/final-validation.cjs
```

### What it does

1. Reads `sidebar-validation.json`.
2. Re-reads `config/sidebar.tsx` to get the current link count.
3. Checks that every remaining sidebar link points to a valid file and that no previously broken links remain.
4. Prints a pass/fail summary for each acceptance criterion.

### Reading the output

```
🔗 FINAL VALIDATION #127

📊 STATISTICS
✅ Total component files available: 2
✅ Links in sidebar (before): 5
✅ Links in sidebar (after): 2
✅ Broken links removed: 3
✅ Reduction: 3 links removed (60%)

📋 ACCEPTANCE CRITERIA
  ✅ All sidebar links point to existing files
  ✅ No broken links remain in sidebar
  ✅ Components section not empty (2 items)
  ✅ Sidebar structure preserved (type: 'group')
  ✅ Validation scripts created

🎉 ALL ACCEPTANCE CRITERIA MET!
```

If any criterion fails, the script exits with code `1` and prints `⚠️ VALIDATION FAILED`.

## Recommended workflow

Run the scripts in this order before every pull request:

```bash
# 1. Validate sidebar entries against the filesystem
pnpm validate:sidebar

# 2. Start the dev server (keep it running)
pnpm dev

# 3. In a second terminal, check live links
pnpm check:links

# 4. Generate the final acceptance summary
node scripts/final-validation.cjs
```

All three commands should complete without errors before you open a PR.
