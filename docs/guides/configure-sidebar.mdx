# Configure the Sidebar Navigation

This guide explains how the docs sidebar navigation is configured in this repo, how to add or reorder entries, and which validation scripts to run before opening a PR.

## Where the sidebar is defined

The sidebar navigation is configured in `config/sidebar.tsx`.

That file exports a `sidebarNav` array. Each entry in the array is a **section** (top-level heading in the sidebar) that can either:

- link to a page directly (via `href`), or
- expand to show a list of child pages (via `pages`)

## Sidebar structure (sections and pages)

`config/sidebar.tsx` defines a few TypeScript types that describe the shape:

- **`SidebarPage`**: a single leaf item (usually a docs page)
- **`SidebarSection`**: a top-level sidebar group that can contain `pages`

Common fields you'll edit:

- **`title`**: the label shown in the UI
- **`href`**: the URL path for the page (should start with `/docs/…`)
- **`pages`**: an ordered list of child pages for that section
- **`defaultOpen`**: whether the section is expanded by default
- **`icon`**: an optional React node (typically a Lucide icon)

### How paths map to files

Docs pages live under `docs/` and are built by Contentlayer.

The URL for a page is derived from its path under `docs/`:

- **File**: `docs/guides/add-docs-page.mdx`
- **URL / href**: `/docs/guides/add-docs-page`

When you add a new entry to the sidebar, the `href` must match the file path under `docs/` **without** the `.mdx` extension.

## Add a new sidebar entry

### 1. Create the docs page

Create a new MDX file under `docs/` in the appropriate folder (for guides, use `docs/guides/`).

Make sure your page includes frontmatter at the top:

```yaml
---
title: Your Page Title
description: A short description used for metadata and search.
---
```

### 2. Register it in `config/sidebar.tsx`

Find the section you want (for guides, the `Guides` section) and add a new page object to `pages`:

```ts
pages: [
  { title: 'Contributing', href: '/docs/guides/contributing' },
  { title: 'Editor Setup', href: '/docs/guides/editor-setup' },
  { title: 'Add a Docs Page', href: '/docs/guides/add-docs-page' },
  {
    title: 'Configure the Sidebar Navigation',
    href: '/docs/guides/configure-sidebar',
  },
  // ...
];
```

Use:

- a **human title** for `title`
- a **kebab-case path** for `href`

## Reorder entries

Ordering is controlled purely by array order in `config/sidebar.tsx`:

- the order of sections in `sidebarNav`
- the order of items in each section's `pages` array

To reorder, move the corresponding object up or down in the array.

## Validation scripts you should run

### Build content (required)

Run the content build to ensure MDX frontmatter and content compile:

```bash
pnpm build:content
```

### Validate internal links (required)

This repo includes a link checker script:

```bash
pnpm check:links
```

Note: `scripts/check-links.cjs` requests pages from a local server (`http://localhost:3000`). If you see connection errors, start the dev server in another terminal:

```bash
pnpm dev
```

### Validate sidebar entries (recommended when editing navigation)

If you edit the sidebar (or component entries referenced by sidebar/TOC), run:

```bash
pnpm validate:sidebar
```

This script checks that component links in `config/sidebar.tsx` and `config/toc.tsx` match real files under `docs/components/`, and it updates `sidebar-validation.json`.

## Verify the page renders

For a quick local sanity check:

1. Start the dev server:

```bash
pnpm dev
```

2. Open the page at its expected URL (based on file path). For example, `docs/guides/contributing.mdx` renders at `/docs/guides/contributing`.

If the page loads and the content build/link checks pass, the guide is ready for PR review.
